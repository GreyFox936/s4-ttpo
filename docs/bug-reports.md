# Баг-репорты F-Bank

## DEF-001 — В форме рублёвого перевода принимается отрицательная сумма после ввода валидного номера карты

**Статус:** Open  
**Severity:** Critical  
**Priority:** High  
**Frequency:** Always  
**Компонент:** Форма перевода денежных средств  
**Связанный тест-кейс:** TC-004  
**Связанный автотест:**   `test_defect_negative_transfer_amount_must_be_rejected`  


### Окружение

- Приложение: F-Bank
- Браузер: Google Chrome / Chromium
- ОС: Windows
- Запуск: `python3 -m http.server 8000`
- URL: `http://localhost:8000/?balance=5000&reserved=0`

### Предусловия

1. Открыта главная страница сервиса.
2. Выбран рублёвый счёт.
3. Введён валидный номер карты `1111222233334444`.

### Шаги воспроизведения

1. Открыть `http://localhost:8000/?balance=5000&reserved=0`.
2. Выбрать счёт `Рубли`.
3. Ввести номер карты `1111222233334444`.
4. Ввести сумму перевода `-1000`.
5. Проверить состояние формы перевода.

### Фактический результат

Приложение отображает кнопку `Перевести` и позволяет отправить перевод с отрицательной суммой.

### Ожидаемый результат

Приложение не должно разрешать перевод отрицательной суммы. Кнопка `Перевести` не должна отображаться, либо должна отображаться ошибка валидации.

### Дополнительная информация

Отрицательная сумма не является корректной суммой перевода денежных средств. Дефект относится к проверке БЛ и проверке пользовательского ввода.

### Рекомендация

Добавить проверку, что сумма перевода является положительным числом больше нуля.



## DEF-002 — Рублёвый перевод блокируется, когда сумма перевода с комиссией равна доступной сумме

**Статус:** Open  
**Severity:** Major  
**Priority:** High  
**Frequency:** Always  
**Компонент:** Расчёт доступности рублёвого перевода  
**Связанный тест-кейс:** TC-005  
**Связанный автотест:**   `test_defect_transfer_allowed_when_amount_plus_commission_equals_available_balance`  


### Окружение

- Приложение: F-Bank
- Браузер: Google Chrome / Chromium
- ОС: Windows
- Запуск: `python3 -m http.server 8000`
- URL: `http://localhost:8000/?balance=1100&reserved=0`

### Предусловия

1. Сервис открыт с параметрами `balance=1100&reserved=0`.
2. Доступная сумма равна `1100 ₽`.
3. Выбран рублёвый счёт.
4. Введён валидный номер карты.

### Шаги воспроизведения

1. Открыть `http://localhost:8000/?balance=1100&reserved=0`.
2. Выбрать счёт `Рубли`.
3. Ввести номер карты `1111222233334444`.
4. Ввести сумму перевода `1000`.
5. Проверить состояние формы перевода.

### Фактический результат

Приложение показывает сообщение `Недостаточно средств на счете` и не отображает кнопку `Перевести`.

### Ожидаемый результат

Приложение должно отобразить кнопку `Перевести`, потому что:

- доступная сумма: `1100 ₽`;
- сумма перевода: `1000 ₽`;
- комиссия 10%: `100 ₽`;
- сумма с комиссией: `1100 ₽`.

Сумма с комиссией не превышает доступную сумму, поэтому перевод должен быть возможен.


## DEF-003 — Поле номера карты принимает 17 цифр вместо ровно 16

**Статус:** Open  
**Severity:** Medium  
**Priority:** Medium  
**Frequency:** Always  
**Компонент:** Поле номера карты  
**Связанный тест-кейс:** TC-006  
**Связанный автотест:** `test_defect_card_number_must_not_accept_more_than_16_digits`  
  

### Окружение

- Приложение: F-Bank
- Браузер: Google Chrome / Chromium
- ОС: Windows
- Запуск: `python3 -m http.server 8000`
- URL: `http://localhost:8000/?balance=5000&reserved=0`

### Предусловия

1. Открыта главная страница сервиса.
2. Выбран любой счёт, например `Рубли`.
3. Отображается поле `Номер карты`.

### Шаги воспроизведения

1. Открыть `http://localhost:8000/?balance=5000&reserved=0`.
2. Выбрать счёт `Рубли`.
3. Ввести в поле номера карты 17 цифр: `11112222333344445`.
4. Проверить значение поля.

### Фактический результат

Поле принимает 17 цифр и отображает значение в формате:

```text
1111 2222 3333 4444 5
```

### Ожидаемый результат

Поле номера карты должно принимать ровно 16 цифр. При вводе 17-й цифры она должна игнорироваться, либо должна отображаться ошибка валидации.

### Дополнительная информация

В инструкции указано что номер карты должен состоять ровно из 16 цифр.

### Рекомендация

Ограничить ввод номера карты 16 цифрами.

## DEF-004 — Валютные счета доступны для выбора, но доступность перевода рассчитывается по рублёвому балансу из URL

**Статус:** Open  
**Severity:** Medium  
**Priority:** Medium  
**Frequency:** Always  
**Компонент:** Выбор счёта и расчёт доступности перевода  
**Связанный тест-кейс:** TC-007  
**Связанный автотест:** `test_defect_currency_transfer_must_use_selected_account_balance`  

### Окружение

- Приложение: F-Bank
- Браузер: Google Chrome / Chromium
- ОС: Windows
- Запуск: `python3 -m http.server 8000`
- URL: `http://localhost:8000/?balance=0&reserved=0`

### Предусловия

1. Открыта главная страница сервиса.
2. В интерфейсе отображается долларовый счёт с балансом `100 $` и резервом `0 $`.
3. Рублёвый баланс в URL равен `0`.

### Шаги воспроизведения

1. Открыть `http://localhost:8000/?balance=0&reserved=0`.
2. Выбрать счёт `Доллары`.
3. Ввести номер карты `1111222233334444`.
4. Ввести сумму перевода `50`.
5. Проверить состояние формы перевода.

### Фактический результат

Приложение показывает сообщение `Недостаточно средств на счете` и не отображает кнопку `Перевести`.

### Ожидаемый результат

Так как в интерфейсе выбран долларовый счёт с балансом `100 $` и сумма перевода `50 $` не превышает доступную сумму, приложение должно разрешить перевод.

Если валютные переводы не поддерживаются, интерфейс должен явно сообщать об этом или не позволять выбирать валютные счета для сценария перевода.

### Дополнительная информация

Дефект связан с несогласованностью интерфейса и бизнес-логики: пользователь выбирает валютный счёт, но расчёт доступности операции выполняется не по выбранному счёту, а по рублёвым параметрам URL.

### Рекомендация

Согласовать интерфейс и логику:

1. либо рассчитывать доступность перевода по выбранному счёту;
2. либо сделать валютные счета недоступными для перевода;
3. либо отображать сообщение, что валютные переводы не поддерживаются.
